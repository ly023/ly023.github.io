ResizeObserver API

发表于 2019-07-08 | 更新于 2019-08-07 | 分类于 JavaScript | 评论数： | 阅读次数：

文档：https://drafts.csswg.org/resize-observer/#resize-observer-interface

简介

当我们需要知道一个元素的大小变化时，需要监听window.resize事件。由于reize事件是高频触发事件，所以很容易导致性能问题。而且resize事件会监听每个元素的大小变化，无法具体到某个元素的变化。而ResizeObserver API可以监听某个DOM节点的变化并触发对应的回调事件。

基本用法

new ResizeObserver(callback)：创建 ResizeObserver 对象

observe(target)：添加待观察元素

unobserve(target)：将元素从观察目标列表中移除

disconect()：清空观察目标列表以及激活目标（目标元素发生变化但未执行回调）列表

entry.contentRect是一个对象，包含了节点的位置属性，如width, height, left, right, bottom, left, x, y等。

width：元素的content width

height：元素content height

top：元素的padidng-top

left：元素的padding-left

right：left + width

bottom: top + height

x：与top相同

y：与left相同

处理

  • 元素的插入、删除会触发回调
  • 元素的display设为none时，会触发回调
  • 内联非替换元素（non-replaced inline element）不会触发回调
  • 由 CSS transform 引发的元素变化不会触发回调

兼容性

只有 Chrome 64+ 支持 Resize Observer，可以使用polyfill：resize-observer-polyfill
